## 한 줄 소개 (Headline)

도메인에 독립적인 코어 위에서 멀티 에이전트 워크플로우를 조립하는 자동화 플랫폼을 설계·구현. LLM 구조화 출력 파싱, 재시도, 점진적 저하 등 프로덕션 신뢰성 엔지니어링에 강점.

## 프로젝트 요약 (Project Summary)

**AI Agent Core — 플러그인 기반 멀티 에이전트 자동화 프레임워크** · Python, FastAPI, asyncio
GitLab MR 코드리뷰·문서 자동화·CI 에러 자동수정을 하나의 웹훅 게이트웨이 뒤에서 처리하는 자가호스팅 오케스트레이션 엔진. "코어는 비즈니스 도메인을 모른다"는 원칙으로 코어와 서비스 로직을 완전 분리, 코어 수정 없이 신규 서비스를 플러그인으로 확장 가능하게 설계.

---

## 핵심 성과 (Bullets)

- **도메인 무지(domain-agnostic) 코어 아키텍처 설계**: 웹훅 게이트웨이·오케스트레이터·에이전트 생명주기를 비즈니스 로직과 분리하여, 신규 자동화 서비스를 **코어 코드 변경 0줄**로 추가 가능한 플러그인 SPI 구축(`ServicePlugin`/`BaseAgent` 추상화).

- **LLM 프로바이더 추상화 계층 구현**: Gemini·OpenAI를 단일 인터페이스(`BaseLLMProvider`) 뒤로 통합하고 레지스트리로 중앙 관리, **설정 한 줄로 모델 벤더 교체** 가능. 동기 SDK를 `asyncio.to_thread`로 감싸 이벤트 루프 블로킹 제거.

- **LLM 구조화 출력 파싱 견고화**: 모델이 코드펜스/잡텍스트를 섞어 반환하는 케이스를 방어하는 JSON 파서를 구현(펜스 제거 → 부분 블록 추출 폴백)하여, 비정형 응답으로 인한 파이프라인 실패를 차단. 단위 테스트로 회귀 고정.

- **일시 장애 복원력 확보**: 외부 API(LLM·GitLab) 호출에 지수 백오프 + 지터 재시도 데코레이터를 적용하고, 키 미구성 시 단계만 건너뛰고 흐름은 완주하는 **점진적 저하(드라이런)** 를 구현하여 로컬·CI에서 외부 의존성 없이 전체 파이프라인 검증 가능.

- **멀티 에이전트 파이프라인 오케스트레이션**: 에이전트별 `asyncio.wait_for` 타임아웃, 취소 플래그·에러 전파 기반 조기 중단, 메인 흐름을 막지 않는 백그라운드 서브에이전트 등 안전장치를 갖춘 비동기 실행 엔진 구현.

- **코드리뷰 플러그인 end-to-end 구현**: GitLab MR 디프 수집 → 리뷰 대상 필터링 → LLM 분석(severity/category 구조화) → MR 코멘트 자동 게시까지 동작. "노이즈보다 정확도" 원칙의 프롬프트로 저가치 코멘트 억제.

- **테스트 주도 검증**: 가짜 프로바이더/클라이언트 주입으로 외부 호출 없이 파이프라인을 검증하는 테스트를 포함해 **총 14개 단위 테스트**(정상·에러중단·취소·타임아웃·파싱·재시도·E2E) 통과. 웹훅은 `hmac.compare_digest` 상수 시간 비교로 서명 검증.

## 면접 대비 — 예상 질문 & 답변 포인트

- **"왜 LangGraph를 안 쓰고 직접 만들었나?"** → 우리 사용 맥락("웹훅 → 자동 라우팅 → 사내 서비스 실행")은 조건부 그래프 런타임이 과했고, 코어를 얇게 유지하는 게 우선. 다만 분기·휴먼인더루프가 필요해지면 파이프라인을 그래프로 승격하는 게 다음 과제임을 안다(트레이드오프 인지).

- **"LLM 출력이 깨지면?"** → 코드펜스 제거 + 부분 JSON 추출 폴백 + 재시도, 최종 실패 시 파이프라인을 죽이지 않고 에러로 기록(graceful degradation). 신뢰성은 모델이 아니라 주변 엔지니어링에서 나온다.

- **"확장성은?"** → 새 서비스 = 폴더 하나 추가(plugin/agents). 코어는 `pkgutil`로 런타임 자동 탐색·등록. 코어 수정 없음이 설계 불변식.

- **한계 솔직히 말하기**: 상태 영속화·체크포인트, 관측성(트레이싱), 조건부 그래프, 휴먼인더루프 부재 — 로드맵으로 관리 중(리포지토리 프로바이더 추상화 → GitHub 어댑터 → DB 영속화).
